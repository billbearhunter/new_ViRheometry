# ViRheometry

**Virtual Rheometry via MPM Simulation** — Estimates Herschel-Bulkley rheological parameters (η, n, σ_y) from real dam-break flow experiments using CMA-ES optimization with a Mixture-of-Experts surrogate model.

## Installation

```bash
chmod +x build.sh
./build.sh
```

The script performs the following steps automatically:

| Step | Content |
|------|---------|
| 1 | Homebrew / apt system packages (cmake, eigen, opencv, python3.11) |
| 2 | libcmaes build from source (non-critical) |
| 3 | All Python packages (torch, taichi, gpytorch, cma, scikit-learn, …) |
| 4 | C++ builds: `GLRender3d` and `cpp_marching_cubes` |
| 5 | Import verification for all packages |

Options:

```bash
./build.sh --py-only    # Python packages only (skip C++ build)
./build.sh --cpp-only   # C++ build only (skip Python)
```

## Typical Workflow

```
Real experiment
      │
      ▼
① Calibrate camera  (Calibration/pipeline.py)
      │
      ▼
② Extract flow distances from photos  (Calibration/extract_flow_distance.py)
      │
      ▼
③ Optimize HB parameters with CMA-ES  (Optimization/optimize_1setup.py)
      │
      ▼
④ Verify: simulate with estimated params and compare  (Simulation/main.py)
      │
      ▼
⑤ Fit rheometer data for ground truth  (FlowCurve/hb_fit.py + flowcurve.py)
```

### FlowCurve

#### `hb_fit.py` — Herschel-Bulkley curve fitting

Fits σ = K · γ̇ⁿ + σ_Y to Anton Paar rheometer CSV data.

```bash
python3 hb_fit.py --file <CSV> [--range START END]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--file` | required | Path to rheometer CSV (UTF-16, header at row 5) |
| `--range START END` | `5 19` | Row index range for fitting |

**Example:**
```bash
python3 hb_fit.py --file Rheo_Data/tonkatsu_20230113_2000_23C.csv
python3 hb_fit.py --file Rheo_Data/Lotion_20230114_1204_23C.csv --range 3 18
```

**Output:** Prints K, n, σ_Y with ±error and R² to stdout.

#### `flowcurve.py` — Flow curve visualization

Plots experimental data with one or more HB model overlays.

```bash
python3 flowcurve.py --file <CSV> --est η n σ_Y [--est η n σ_Y ...] --out <PDF> [--extent_y Y_MIN Y_MAX]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--file` | required | Rheometer CSV file |
| `--est η n σ_Y` | required | HB parameters (repeat for multiple curves) |
| `--out` | required | Output PDF path |
| `--extent_y Y_MIN Y_MAX` | `1e0 1e2` | Y-axis range |

**Example:**
```bash
# Single estimated curve
python3 flowcurve.py \
    --file Rheo_Data/tonkatsu_20230113_2000_23C.csv \
    --est 208.35 0.306 95.26 \
    --out figs/Tonkatsu.pdf

# Ground truth vs two estimates
python3 flowcurve.py \
    --file Rheo_Data/Lotion_20230114_1204_23C.csv \
    --est 2.711 0.6537 8.404 \
    --est 2.929 0.621 8.393 \
    --out figs/lotion_comparison.pdf \
    --extent_y 1 200
```

### Calibration

#### `pipeline.py` — Camera calibration

Full pipeline: ChArUco board calibration → DLT fine-tuning → outputs `camera_params.xml` and background mask images.

```bash
python3 pipeline.py \
    --calib_img <IMG> \
    --target <config_00.png> \
    [--bg_img <IMG>] \
    [--cube_alpha 0.7] \
    [--skip_calib --theta0 "..."]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--calib_img` | required | Photo of the ChArUco board |
| `--target` | required | Reference simulation snapshot (`config_00.png`) |
| `--bg_img` | same as `--calib_img` | Background photo for rendering |
| `--cube_alpha` | `0.7` | Transparency of the wireframe cube overlay |
| `--skip_calib` | — | Skip ChArUco step; provide `--theta0` manually |
| `--theta0` | — | Initial camera parameter string (when `--skip_calib`) |

**Output** (saved in the same directory as `--calib_img`):
- `camera_params.xml` — intrinsic + extrinsic camera parameters
- `Background_mask.png` — binary mask of the fluid region
- `Background.png` — rendered background

**Example:**
```bash
cd Calibration
python3 pipeline.py \
    --calib_img ../data/ref_Tonkatsu_6.7_3.5_1/IMG_7806.JPG \
    --target    ../data/ref_Tonkatsu_6.7_3.5_1/config_00.png
```

> **Note:** The directory containing `--calib_img` must also contain `settings.xml`:
> ```xml
> <setup W="6.3" H="4.6" ... />
> ```

#### `extract_flow_distance.py` — Extract flow distances from experiment photos

Processes a series of `config_XX.png` images to extract the flow front position at each time step. Outputs a CSV/JSON of 8 flow distance values used as input to CMA-ES.

```bash
python3 extract_flow_distance.py --dir <DATA_DIR> [options]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--dir` | required | Directory containing `config_*.png`, `settings.xml`, `camera_params.xml` |
| `--settings` | auto | Override `settings.xml` path |
| `--camera-xml` | auto | Override `camera_params.xml` path |
| `--images` | `config_*.png` | Override image glob pattern |
| `--threshold` | `128` | Binarization threshold |
| `--foreground` | `black` | Fluid color: `black` or `white` |
| `--kernel-size` | `3` | Morphological kernel size |
| `--opening-only` | — | Apply opening only (skip closing) |
| `--no-keep-largest` | — | Disable largest-component filtering |
| `--flow-direction DX DZ` | `1.0 0.0` | Flow direction vector |
| `--box-front` | auto | Override container front wall position |
| `--monotonic` | — | Enforce monotonically increasing distances |
| `--unit` | `cm` | Output unit: `m`, `cm`, or `mm` |
| `--output-csv` | `<dir>/flow_distances.csv` | Output CSV path |
| `--output-json` | `<dir>/flow_distances.json` | Output JSON path |
| `--debug-dir` | — | Save debug images for each step |
| `--print-dis1` | — | Print `-dis1` argument string for direct use with `optimize_1setup.py` |

**Example:**
```bash
cd Calibration
python3 extract_flow_distance.py \
    --dir ../data/ref_Tonkatsu_6.7_3.5_1 \
    --foreground black \
    --monotonic \
    --unit cm \
    --print-dis1
```

**Output example** (with `--print-dis1`):
```
-dis1 0.409 1.010 1.421 1.617 1.730 1.816 1.890 1.954
```
→ Copy this directly into the `optimize_1setup.py` command.

### Simulation

#### `main.py` — Run MPM simulation and compare with reference

Runs one MPM simulation with given HB parameters, renders PNG snapshots, and generates pixel-level difference images against a reference experiment.

```bash
cd Simulation
python3 main.py \
    --eta <η> --n <n> --sigma_y <σ_y> \
    --ref <REF_DIR> \
    [--diff_amplify 5.0]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--eta` | required | Viscosity η [Pa·sⁿ] |
| `--n` | required | Power-law index n [-] |
| `--sigma_y` | required | Yield stress σ_y [Pa] |
| `--ref` | required | Reference directory with `config_XX.png`, `settings.xml`, `camera_params.xml` |
| `--diff_amplify` | `5.0` | Brightness amplification for diff images |

**Output** (saved to `results/run_YYYYMMDD_HHMMSS/`):
```
run_20260409_153000/
├── simulation_results.csv        # parameters + displacement values
├── {n}_{eta}_{sigma_y}/
│   ├── config_00.png ~ config_07.png   # rendered snapshots
│   └── *.obj                           # mesh files
└── snapdiff_00.png ~ snapdiff_07.png   # diff vs reference
```

**Example:**
```bash
cd Simulation
# From optimizer result
python3 main.py \
    --eta 4.537 --n 0.9999 --sigma_y 1.007 \
    --ref ../data/ref_Tonkatsu_6.7_3.5_1

# Ground truth verification
python3 main.py \
    --eta 208.35 --n 0.306 --sigma_y 95.26 \
    --ref ../data/ref_Tonkatsu_6.7_3.5_1 \
    --diff_amplify 10.0
```

#### `optimize_1setup.py` — CMA-ES optimization (1 setup)

Estimates (η, n, σ_y) from a **single** dam-break experiment using CMA-ES with the MoE surrogate model.

```bash
cd Optimization
python3 optimize_1setup.py \
    -W1 <W> -H1 <H> \
    -dis1 d1 d2 d3 d4 d5 d6 d7 d8 \
    --moe_dir <DIR> \
    [--strategy topk|threshold|adaptive|all] \
    [--topk N] [--threshold F] [--confidence_threshold F] [--max_experts N] \
    [--sigma0 F] [--popsize N] [--maxiter N] [--seed N] \
    [--compare_strategies]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `-W1` | required | Container width [cm] |
| `-H1` | required | Container height [cm] |
| `-dis1` | required | 8 flow distances [cm] |
| `--moe_dir` | required | Path to MoE model directory |
| `--strategy` | `threshold` | Expert selection: `topk`, `threshold`, `adaptive`, `all` |
| `--topk` | `2` | Number of experts for `topk` strategy |
| `--threshold` | `0.01` | Weight threshold for `threshold` strategy |
| `--confidence_threshold` | `0.7` | Confidence threshold for `adaptive` strategy |
| `--max_experts` | `5` | Max experts for `threshold` strategy |
| `--sigma0` | `0.5` | CMA-ES initial step size |
| `--popsize` | `16` | CMA-ES population size |
| `--maxiter` | `700` | Maximum CMA-ES iterations |
| `--seed` | `42` | Random seed |
| `--compare_strategies` | — | Run and compare all strategies |

**Output** (saved to `result_setup1_<strategy>_<threshold>_<timestamp>/`):
- `result.json` — best (η, n, σ_y) estimate
- `convergence.png` — loss curve
- `cmaes_log.dat` — full CMA-ES log

**Examples:**
```bash
cd Optimization

# Standard run (Top-2 experts)
python3 optimize_1setup.py \
    --moe_dir moe_workspace5 \
    -W1 2.0 -H1 3.5 \
    -dis1 0.409 1.010 1.421 1.617 1.730 1.816 1.890 1.954 \
    --strategy topk --topk 2

# Adaptive gating
python3 optimize_1setup.py \
    --moe_dir moe_workspace5 \
    -W1 2.1 -H1 4.1 \
    -dis1 1.221 3.934 6.944 9.293 10.513 10.951 11.049 11.082 \
    --strategy adaptive --confidence_threshold 0.7

# Compare all strategies at once
python3 optimize_1setup.py \
    --moe_dir moe_workspace5 \
    -W1 2.0 -H1 3.5 \
    -dis1 0.409 1.010 1.421 1.617 1.730 1.816 1.890 1.954 \
    --compare_strategies
```

#### `optimize_2setups.py` — CMA-ES optimization (2 setups)

Estimates (η, n, σ_y) from **two** dam-break experiments simultaneously for higher accuracy. Optionally extends to 3 setups.

```bash
cd Optimization
python3 optimize_2setups.py \
    -W1 <W> -H1 <H> -dis1 d1..d8 \
    -W2 <W> -H2 <H> -dis2 d1..d8 \
    --moe_dir <DIR> \
    [--setup1_dir <DIR>] \
    [--strategy ...] [same CMA-ES options as optimize_1setup.py]
```

All arguments from `optimize_1setup.py` apply, plus:

| Argument | Default | Description |
|----------|---------|-------------|
| `-W2`, `-H2` | required | Setup 2 container dimensions [cm] |
| `-dis2` | required | Setup 2 flow distances (8 values) [cm] |
| `-W3`, `-H3` | `0.0` | Setup 3 dimensions (optional) |
| `-dis3` | `[]` | Setup 3 flow distances (optional) |
| `--setup1_dir` | auto | Reuse Setup 1 result directory from a previous run |

**Examples:**
```bash
cd Optimization

# 2-setup optimization (fresh run)
python3 optimize_2setups.py \
    --moe_dir moe_workspace5 \
    -W1 2.0 -H1 3.5 \
    -dis1 0.409 1.010 1.421 1.617 1.730 1.816 1.890 1.954 \
    -W2 2.4 -H2 2.0 \
    -dis2 0.283 0.490 0.616 0.708 0.782 0.845 0.898 0.946 \
    --strategy topk --topk 2

# Reuse existing Setup 1 result
python3 optimize_2setups.py \
    --setup1_dir result_setup1_adaptive_0.01_20260203_231654 \
    --moe_dir moe_workspace5 \
    -W1 2.0 -H1 3.5 \
    -dis1 0.409 1.010 1.421 1.617 1.730 1.816 1.890 1.954 \
    -W2 2.4 -H2 2.0 \
    -dis2 0.283 0.490 0.616 0.708 0.782 0.845 0.898 0.946 \
    --strategy adaptive --confidence_threshold 0.9
```

#### `soft_interpolate.py` — Soft gating module (library)

Not a standalone script. Imported by `optimize_1setup.py`. Provides GMM-based soft expert weighting as an alternative to hard Top-K selection.

Key functions:
- `build_phi(y, W, H)` — Build gating feature vector from flow distances and geometry
- `get_soft_weights(gate_dict, phi, topk, confidence_threshold)` — Compute GMM soft weights
- `compare_hard_vs_soft(...)` — Compare predictions of both strategies

Not a standalone script. Imported by `optimize_1setup.py` when `--compare_strategies` is used. Generates multi-panel comparison plots of Hard-TopK vs Soft-GMM gating.

## Notes

### HB parameter units

All physical parameters follow the **CGS unit system** used internally by the MPM solver:

| Parameter | Symbol | Unit (CGS) |
|-----------|--------|-----------|
| Viscosity | η | Pa·sⁿ (× 0.1 → CGS) |
| Power-law index | n | dimensionless |
| Yield stress | σ_y | Pa (× 0.1 → CGS) |
| Width / Height | W, H | cm |
| Flow distance | dis | cm |

### MoE model (`moe_workspace5`)

The pre-trained model contains 101 neural network experts with GMM-based gating. It predicts 8 flow distance values from (W, H, η, n, σ_y) in milliseconds, enabling fast CMA-ES evaluation without running full MPM simulations.

### `lcmaes` (non-critical)

`lcmaes` is only used in `libs/cmaes.py`, which is a legacy alternative implementation. The main optimization scripts (`optimize_1setup.py`, `optimize_2setups.py`) use `import cma` (pycma) and work without lcmaes.

### Taichi GPU backend

The MPM simulator uses Taichi with GPU by default. To switch to CPU (for machines without GPU), edit `Simulation/main.py`:

```python
# GPU (default)
ti.init(arch=ti.gpu, offline_cache=True, default_fp=ti.f32, default_ip=ti.i32)

# CPU fallback
ti.init(arch=ti.cpu, offline_cache=True, default_fp=ti.f32, default_ip=ti.i32)
```
